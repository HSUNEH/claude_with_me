Git 커밋 및 푸쉬를 수행해주세요.

1. `git status`로 변경사항 확인
   - 현재 디렉토리가 git 저장소가 아닌 경우, 하위 폴더에서 git 저장소를 찾아 작업
2. `git diff`로 변경 내용 확인
3. 변경사항이 있으면:
   - 모든 파일 스테이징 (`git add -A`)
   - 변경 내용을 분석하여 적절한 커밋 메시지 작성
   - 커밋 메시지는 한국어로, 간결하게 작성
   - 커밋 실행
   - `git push origin main`으로 푸쉬
4. 결과 요약 출력

**중요: 디렉토리 복귀 규칙**
- git 작업을 위해 다른 디렉토리로 이동한 경우, 작업 완료 후 반드시 원래 디렉토리로 복귀할 것
- `cd`로 이동하기 전에 원래 경로를 기억하고, 모든 작업이 끝나면 `cd 원래경로`로 돌아올 것
- 가능하면 `cd` 대신 `git -C <경로>` 옵션을 사용하여 디렉토리 이동 없이 작업할 것
