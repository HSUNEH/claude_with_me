# 계획 관리 스킬

> 모든 개발 작업은 **계획 수립 → 승인 → 실행** 순서를 따른다.
> 이 스킬은 계획 수립 단계를 자동화한다.

## 실행 흐름

```
1. 사용자가 작업 지시
       │
       ▼
2. 계획 수립 (이 스킬)
   ├── 요구사항 분석
   ├── 기술 조사
   └── 구현 전략 설계
       │
       ▼
3. 3문서 생성 → docs/plans/{작업명}/
   ├── PLAN.md         계획서
   ├── CONTEXT.md      맥락 노트
   └── CHECKLIST.md    체크리스트
       │
       ▼
4. 사용자 검토 & 승인 요청
       │
       ▼
5. 승인 후 → 작업 시작
```

## 3문서 작성 규칙

### 문서 1: PLAN.md (계획서)

**목적**: 뭘 만들 건지, 처음부터 끝까지

```markdown
# [작업명] 계획서

## 개요
- 작업 목적 (한 줄 요약)
- 예상 범위 (영향받는 파일/모듈)
- 예상 소요 (단계 수)

## 현재 상태 분석
- 기존 코드 구조
- 변경이 필요한 부분

## 구현 계획
### Phase 1: [단계명]
- 구체적 작업 내용
- 예상 변경 파일

### Phase 2: [단계명]
...

## 기술 선택
- 사용할 라이브러리/패턴
- 선택 이유 (간략)

## 리스크
- 예상되는 문제점
- 대응 방안
```

### 문서 2: CONTEXT.md (맥락 노트)

**목적**: 왜 이렇게 결정했는지, 자료 위치

```markdown
# [작업명] 맥락 노트

## 결정 기록
| 결정 사항 | 선택지 | 최종 선택 | 이유 |
|-----------|--------|-----------|------|

## 참조 자료
- 관련 문서/URL
- 참고한 코드 위치

## 주요 제약 조건
- 기술적 제약
- 비즈니스 제약

## 사용자 요구사항 원문
> (사용자가 지시한 내용 그대로 기록)
```

### 문서 3: CHECKLIST.md (체크리스트)

**목적**: 뭘 끝냈고, 뭐가 남았는지 추적

```markdown
# [작업명] 체크리스트

## 진행 상태: 🔴 시작 전 | 🟡 진행 중 | 🟢 완료

## 작업 목록
- [ ] Phase 1: [단계명]
  - [ ] 세부 작업 1
  - [ ] 세부 작업 2
- [ ] Phase 2: [단계명]
  - [ ] 세부 작업 1

## 품질 체크
- [ ] 에러 처리 적용
- [ ] 보안 검토 완료
- [ ] 테스트 작성/통과
- [ ] 코드 리뷰 포인트 확인

## 변경 로그
| 시간 | 파일 | 변경 내용 | 상태 |
|------|------|-----------|------|
```

## 문서 저장 위치

```
docs/plans/{작업명}/
├── PLAN.md
├── CONTEXT.md
└── CHECKLIST.md
```

- `{작업명}`은 kebab-case로 (예: `user-auth`, `api-refactor`)
- 한 작업 = 한 폴더, 문서 3개가 항상 세트

## 중요 규칙

1. **계획 먼저**: 코드 한 줄 쓰기 전에 3문서부터 생성
2. **승인 필수**: 문서 생성 후 반드시 사용자에게 검토 요청
3. **실시간 갱신**: 작업 중 CHECKLIST.md는 계속 업데이트
4. **맥락 보존**: 새 세션에서도 docs/plans/ 읽으면 이어서 작업 가능
